What is CSS

  • CSS stands for Cascading Style Sheets
  • CSS describes how HTML elements are to be displayed on screen
  • External stylesheets are stored in CSS files

HTML and CSS relation

In order to understand what is the relation between HTML and CSS () in any website, let's have look at following image:

HTML gives our website structure while CSS is taking care of the presentation layer. We will show that on the example.

As we said in the previous lesson our paragraph and div can also have atributes, one of the most common is a class name. Class name allows us to apply CSS on given HTML element, let's add some classes to our tags.


<h1 class="heading"> Our heading (aka title)</h1>
<p class="summary">Some paragraph, with some text </p>
<div class="details">And div which stand for divider, it's meant to be used as container of a data.</div>

Add new tag for styles within the <head></head> tags.


<style>

</style>

Now we can add styles to our website. Let's add some styles to our heading


<style>
.heading{
    font-size: 100px;
}  
</style>

Refresh our page.

This is a heading :) How it works? Basically it says to our browser - if you find style for a given class name, apply given font size.

Using CSS we can change size,color, margins and even transform almost anything on our website.

Let's change color of our paragraph and add some border to our div.

Note: You don't have to remember all CSS properties now, we will learn them in a future. 


.summary{
    color: blue;
}
.details{
    border: solid 1px red;
}

Note: Remember to add above classes in between <style></style> tags!

As you can see we are applying CSS only to dedicated elements, changing color of our paragraph doesn't change the color of our heading.

Good to know: You should learn using Dveloper Tools as soon as possible - this will make your work waaaaay easier.

Developer console which is part of a Developer Tools mechanism built in all modern web browsers is a powerfull tool. It allows you to:

  • preview styles applied to our element
  • check from where given style is coming from
  • modify styles "on the fly"

Note:Changes you are making in the console are not permanent! They will disappear once you refresh the web page.

External stylesheet file

As you can see, we are writing an CSS inside our index.html file - this is not a good practice. Lets move styles to style.css file which we crated at the begining.

Note! You don't need <style> </style> tags when you're writing in separate stylesheet file so remove it.

No we have to somehow connect our style file with index.html file, we will use it using tag which will be placed in between tags.

<link type="text/css" rel="stylesheet" href="style.css" />

But our styles doesn't work as they used to , why ? Let's check our console.

Our page cannot load styles.css file becasue we made a typo, we used style.css instead of styles.css (note plural vs singular). Let's fix it:

<link type="text/css" rel="stylesheet" href="styles.css" />

This is why you need to get familiar with the console - it shows you all error you can possibly make.
